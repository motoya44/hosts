あなたには今からドメインを分類してブラウザゲームをブロックするホストファイルの整理を手伝ってもらいます。
もしこの指示よりもっといい分類の仕方を思いついたらその都度提案してください。

-返信の原則
・コードはコードブロックで返す。

-ドメインの原則
・渡したコードのドメインは勝手に削除・変更・追加してはいけない。絶対抜けがないように。
・あなたが追加したいと思うときは必ず提案して了解を得ろ。yy
・あなたが推測して生み出したドメインを追加しようとしてはいけない。
・閉鎖／休止／売却済み（ドメインパーキング・転売ページ等）のドメインは提案しない。
・ドメインは重複禁止。ジャンルを跨いだ二重掲載は認めない。
・ワイルドカード禁止。
・サブドメインは別管理。(例：example.com と www.example.com は別)。
・ドメインにはブランド名／用途など日本語(英語固有名詞は可／半角英字+日本語の混在可)で説明も加えて(短句推奨)。
(例：127.0.0.1 poki.com #説明 )
・punycode表記で小文字統一

-見出しの原則
・見出しの中にドメインを置く。
・分類するときは大きな見出しから考えていく。
・適切な見出しが見つからないなら新しく提案して。
・(提案) とコメントつけて[ ex: #--- 日本のゲームディストリビューション --- (提案) ]
・●●には言語名を入れて
・ローカライズは国別TLDもしくはサブドメイン
・見出しの中が存在しないなら空見出しは作らない(ローカライズやキャッシュ／ブランド独自のCDNミラが存在しない場合がある。)
・---は必ずブランド名を囲む
・ブランド性がないかつドメインが一つだけの場合は「--- (ブランド名) ---」を省略して直接 -----　で囲んだ見出しの下に置く。
・本体だけの場合は「- 本体 -」を省略して 「--- (ブランド名) ---」の下に直接ドメインを置いていい。
(例:
#--- (ブランド名) ---
127.0.0.1 example.com
)

-カテゴライズの優先順位(上から優先する)
①CDNが第三者のものを使ってる場合は「第三者CDN」にのみ分類、ブランド向けにカスタムした第三者CDNも含む。汎用広告ネットワークも同様
②Githubページはまとめる。
③一つの企業、ブランドでまとめることを最優先。同じブランドが複数ジャンルに展開している場合ブランド配下でジャンル分け
④サイトのブランドの教育・脳トレ・ガールズ要素が強ければ「教育/脳トレ/タイピング/ガールズ/その他子供向け」に
⑤2Dio系は「2Dio系」優先

基本的な構造は　(

# ==========================
# (数字). 大ジャンル
# ==========================
# ----- (中ジャンル) -----
# --- (ブランド名) ---

(ccTLDを先、ついで言語サブドメイン。地域コードは ISO 3166/639 を尊重)
# - キャッシュ／ブランド独自のCDNミラ -
(cdn → assets → img → video → その他の順番、必要な時だけ細分化)
# - 広告配信 -
# - ゲームディストリビューション -

# ----- (中ジャンル) -----
# --- (ブランド名) ---
# - (小ジャンル) -

)
# - (ブランド内のゲームのジャンルなど)
